Getting started

Quiver is a 3 kB reactive state library for building fast, predictable interfaces. It gives you fine-grained signals, memoized derivations, and effects that only re-run when the data they actually read changes — no virtual DOM, no dependency arrays, no stale closures. This guide takes you from an empty folder to a running counter in under five minutes.

Installation

Quiver ships as a single ES module with zero runtime dependencies. Install it from your package manager of choice. The library is fully typed, so TypeScript users get autocompletion and inference out of the box with no extra @types package.

bash 
# with npm
npm install quiver-js

# or with pnpm / yarn / bun
pnpm add quiver-js

Requirements. Quiver targets modern runtimes — Node 18+, Deno 1.30+, or any evergreen browser. It relies on native Proxy and WeakRef, so there is no build step required to try it in a Vite or esbuild project.

Your first store

Everything in Quiver starts with a signal — a single unit of reactive state. Reading a signal inside a tracked scope subscribes that scope to future changes; writing to it schedules every dependent to re-run. Let's build the canonical counter to see the whole cycle.

typescript 
import { signal, computed, effect } from "quiver-js";

// 1. a piece of reactive state
const count = signal(0);

// 2. a value derived from it — recomputed lazily
const doubled = computed(() => count() * 2);

// 3. a side effect that tracks what it reads
effect(() => {
  console.log(`count is ${count()}, doubled is ${doubled()}`);
});

count.set(5); // logs: count is 5, doubled is 10

Notice there is no subscription to unregister and no dependency list to maintain. The effect discovered that it reads count and doubled simply by running once; on the next write it re-runs, re-reads, and re-subscribes automatically.

Reading and writing

A signal is callable. Invoke it with no arguments to read the current value inside a reactive scope, and call .set(next) or .update(fn) to write. Writes are synchronous but effects are flushed on a microtask, so a burst of updates collapses into a single re-render.

Reading outside a tracked scope. Calling a signal at module top level or inside an event handler reads its value but records no dependency. If you expect something to react and it doesn't, you are almost always reading it outside an effect() or computed(). Wrap the read, or use peek() when the non-tracking read is intentional.

Wiring it to the DOM

Quiver is renderer-agnostic, but the companion package @quiver/dom binds signals straight to elements. Follow these steps to get pixels on the screen.

  1. Create a root. Call mount(document.body, App). The root owns a reactive scope and disposes every effect beneath it when unmounted.
  2. Return a template. Components are plain functions that return DOM. Interpolated signals become live bindings — Quiver patches only the exact text node or attribute that changed.
  3. Bind events back to state. Event handlers call .set() or .update(). There is no reducer boilerplate and no action dispatch layer to thread through.
  4. Ship it. Tree-shaking drops anything you didn't import, so a typical counter app weighs in under 4 kB gzipped including the DOM bindings.

Signal options

Both signal() and computed() accept an optional second argument to tune how updates propagate. The table below lists every field, its type, its default, and when you'd reach for it.

Name Type Default Description
equals (a, b) => boolean Object.is Custom equality check. Return true to treat a write as a no-op and skip notifying dependents.
name string undefined A human label surfaced in the devtools timeline and in error traces. Has no runtime cost in production builds.
lazy boolean true For computed values only. When true, the derivation runs on first read rather than eagerly at creation.
eager boolean false Flush dependents synchronously on write instead of batching to a microtask. Use sparingly for animation loops.
onDispose () => void undefined A teardown callback invoked when the owning scope is disposed. Ideal for clearing timers or sockets.
